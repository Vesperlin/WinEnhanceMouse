# WinEnhanceMouse - Windows 11 增强鼠标自定义插件

## I . 目前编写进度与核心能力概览
本仓库目前构建了一个可立即运行的 Windows 11 鼠标增强插件原型。它提供：

-  全局中键菜单（支持滚轮导航、左右键选择、前/后侧键功能）。
-  一层/二层操作队列及顺序执行逻辑。
-  覆盖浏览器、窗口管理、系统控制、文本编辑、应用启动等 60+ 个内置动作。
-  JSON 配置存储与导入/导出、多模式管理、外观自定义。
-  Tkinter 设置面板（模式、按键、AI 配置可视化编辑）。
-  可选的 AI 助手（F9 唤出，基于 OpenAI Chat Completions API）。

> 说明：部分 Windows Snap/置顶等操作基于键盘模拟（Win+Z 布局选择），实际效果取决于当前系统版本及窗口布局。若需要更精确的窗口控制，可扩展动作脚本。

### 环境依赖
- Windows 11 + Python 3.9 及以上版本。
- 需要安装 Microsoft Visual C++ Build Tools（用于部分 Python 包编译）。
- 依赖包：`pynput`, `pyautogui`, `pyperclip`, `psutil`, `openai`, `pillow`。
- 若使用 AI 模式，需要在网络环境下可访问 OpenAI API。

安装依赖：
```powershell
py -3.11 -m venv .venv
.\.venv\Scripts\activate
pip install -e .
```

> 建议在管理员 PowerShell 中运行，以便允许键鼠 Hook 监听。

### 运行与部署
1. **首次启动**（会自动在 `%USERPROFILE%\.winenhancemouse\config.json` 创建配置）：
   ```powershell
   python -m winenhancemouse
   ```
2. 程序启动后常驻系统托盘（命令行窗口保持运行），核心交互：
   - 中键按下：呼出/关闭菜单。
   - 滚轮：在菜单条目间聚焦切换。
   - 左键单击：加入一层队列；左键双击：加入二层队列。
   - 右键：撤销最近加入的动作；若队列为空则退出菜单。
   - 鼠标前侧键（X2）：执行一层后执行二层队列。
   - 鼠标后侧键（X1）：打开设置面板。
   - F9：唤出/关闭 AI 助手聊天窗口。

3. **静默后台运行**：可将入口脚本放入“启动”文件夹，或使用 `pythonw.exe -m winenhancemouse` 隐藏控制台。
4. **终止程序**：在命令行内 `Ctrl+C`，或结束 Python 进程。

### 设置面板使用指南
- 打开方式：在中键菜单模式下按鼠标后侧键，或直接运行 `python -m winenhancemouse` 后按下后侧键。
- “新增/删除模式” 可以管理不同工作流。
- 选中模式后，可在“一层/二层”列表中添加操作：
  - 输入动作 ID（可在提示中查看全部内置动作）。
  - 可附带 JSON 参数（例如 `{"url": "https://www.example.com"}`）。
  - “应用修改” 会立即保存并刷新运行时配置。
- “导入/导出配置” 通过绝对路径读写 JSON 文件，便于备份或共享。

### AI 模式配置
1. 在设置面板右侧勾选“启用 AI”。
2. 填写 OpenAI API Key、模型（默认 `gpt-4o-mini`）以及系统提示词。
3. 点击“应用修改”保存。
4. 运行中按 `F9` 打开 AI 窗口：输入需求后 AI 会给出建议或代码片段，可根据返回结果手动复制到动作参数中。

> 若希望实现“自动修改动作代码”的完全闭环，可在返回结果中嵌入自定义脚本，并通过“运行指定脚本”动作执行。

### 自定义动作扩展
- 在 `winenhancemouse/actions/registry.py` 中新增 Python 函数，并通过 `registry.register("your_action", handler)` 注册。
- 在配置文件或设置面板中引用新的动作 ID，即可立即生效。
- 若需要复杂逻辑，可在动作处理函数内部调用 PowerShell、AutoHotkey、外部 API 等。

### 常见问题 (FAQ)
- **权限不足导致 Hook 无效**：请以管理员权限运行终端或将脚本注册为开机任务时选中“使用最高权限”。
- **Snap 布局编号不匹配**：`window_snap_top`/`window_snap_bottom` 会发送 `Win+Z` 并输入数字，可在设置中调整 `layout` 参数（1-6）。
- **AI 提示窗口无响应**：确认已安装 `openai` 包并配置正确的 API Key，必要时检查代理设置。
- **在非 Windows 环境测试**：依赖的系统调用与按键模拟均针对 Windows，其他平台仅可运行但无法执行动作。

### 核心交互系统：中键菜单 (The Pop-up Menu)
> 中键菜单是本插件的核心交互界面，通过鼠标中键触发，实现快速操作选择。

##### **A. 菜单触发与外观**

| 元素 | 描述 | 交互逻辑 |
|---|---|---|
| 触发键 | 鼠标中键 | 按下中键进入菜单模式；再次按下中键退出菜单（若未选中操作）。 |
| 外观 | 非侵入式弹窗 | 右下角（可配置位置）弹出很小很窄的一条弹窗，无视线遮挡或阻碍。 |
| 导航 | 滚轮滚动 | 上下滚动滚轮选择/聚焦操作，具有类似堆叠滑动的视觉聚焦效果。 |
| 退出菜单 | 单击右键 | 若没有任何操作被选中，单击右键则代表退出菜单模式。 |
| 设置快捷入口 | 后侧键 | 在中键菜单模式下，第一个按下的按键若是后侧键，则直接执行打开程序设置。 |


##### **B. 操作选择与分层执行逻辑**
> 操作被分为两层队列，按照选中顺序执行。

| 队列/动作 | 触发按键/方式 | 描述与执行顺序 |
|---|---|---|
| 一层操作队列 | 单击左键 | 单选/多选：无论当前是否多选模式，单击左键将聚焦的操作添加到第一层执行队列中（选中顺序即执行顺序）。 |
| 执行一层 | 前侧键 | 结束多选，并开始执行第一层队列中的所有操作。 |
| 二层操作队列 | 双击左键 | 将当前聚焦操作添加到第二层执行队列中。此操作不影响其在一层菜单中的状态。 |
| 队列执行顺序 | 系统执行 | 第一层操作全部运行完毕后，才会按照选中顺序执行第二层操作。 |
| 取消选择 | 单击右键 | 无论在哪个队列，单击右键取消当前聚焦项的选择。 |
| 状态标识 | 视觉反馈 | 第一层和第二层被选中后标识不同（如不同颜色/图标），且可叠加，用户必须能清楚区分。 |
